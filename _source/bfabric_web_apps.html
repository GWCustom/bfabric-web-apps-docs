bfabric_web_apps.get_power_user_wrapper(token_data)[source]#

Initializes and returns a Bfabric power user instance configured for a specific environment.

This function retrieves the environment information from the provided token_data and uses it to initialize a Bfabric instance. The configuration file path is determined by the CONFIG_FILE_PATH from the application's configuration.

Parameters:

token_data (dict) -- A dictionary containing token information The key "environment" is used to determine the environment (default is "None" if not specified).

Returns:

A Bfabric instance initialized with the configuration corresponding to the specified environment.

Return type:

Bfabric

bfabric_web_apps.get_static_layout(base_title=None, main_content=None, documentation_content=None)[source]#

Returns a layout with static tabs for Main, Documentation, and Report a Bug. The main content is customizable, while the other tabs are generic.

Parameters:

  • base_title (str) -- The main title to be displayed in the banner.

  • main_content (html.Div) -- Content to be displayed in the "Main" tab.

  • documentation_content (html.Div) -- Content for the "Documentation" tab.

Returns:

The complete static layout of the web app.

Return type:

html.Div

bfabric_web_apps.process_url_and_token(url_params)[source]#

Processes URL parameters to extract the token, validates it, and retrieves the corresponding data. Additionally, it constructs a dynamic job link based on the environment and job ID.

Parameters:

url_params (str) -- The URL parameters containing the token.

Returns:

A tuple containing:

  • token (str): Authentication token.

  • token_data (dict): Token metadata.

  • entity_data (dict): Retrieved entity information.

  • page_title (str): Title for the page header.

  • session_details (list): HTML-formatted session details.

  • job_link (str): Dynamically generated link to the job page.

Return type:

tuple

bfabric_web_apps.submit_bug_report(n_clicks, bug_description, token, entity_data)[source]#

Submits a bug report based on user input, token, and entity data.

Parameters:

  • n_clicks (int) -- The number of times the submit button has been clicked.

  • bug_description (str) -- The description of the bug provided by the user.

  • token (str) -- The authentication token.

  • entity_data (dict) -- The data related to the current entity.

Returns:

A tuple containing two boolean values indicating success and failure status of the submission.

(is_open_success, is_open_failure)

Return type:

tuple
